# 本プロジェクト独自の強み (Unique Features)

本ライブラリは、単にDoclingを使用するだけでなく、実運用に耐えうる多くの改善と機能拡張を施しています。

## 1. 堅牢なセキュリティ設計

Doclingそのものはライブラリですが、これをAPIやCLIとして提供する場合、パス・トラバーサルなどのリスクが生じます。
- **Path Traversal対策**: `Path.resolve()` と `is_relative_to()` を組み合わせた厳格なパス検証により、ホストシステム上の任意のファイルの読み書きを防ぎます。
- **Log Injection対策**: ユーザが提供するファイル名等に含まれる改行文字などをサニタイズし、ログファイルの改ざんを防ぎます。

## 2. 高パフォーマンスな並行処理

標準のDoclingをWebサーバーでそのまま使用すると、メインスレッドがブロックされたり、リソース競合が発生します。
- **Thread-safe設計**: グローバルな `DocumentConverter` インスタンスに対し、`threading.Lock` を導入しました。初期化コストの低減と、スレッドセーフな安全性を両立しています。
- **FastAPIの非同期化**: 重い変換処理を `run_in_threadpool` で実行することで、APIサーバーが他のリクエストに応答できない時間を最小化します。

## 3. 高度な解析機能 (VLM統合)

最新の Docling v2.x 機能を最大限に引き出し、LLM に最適なコンテキストを提供します。
- **数式抽出 (LaTeX)**: `CodeFormulaV2` モデルを統合し、科学技術文書内の数式を LaTeX 形式で正確に抽出します。
- **高精度テーブル**: 複雑なセル結合を保持する HTML テーブル出力。
- **メタデータ統合**: YAML Frontmatter によるドキュメント属性の自動保持。

## 4. 統合されたテスト・検証環境

- **検証スイート**: 実データ（PDF, DOCX, PPTX, XLSX）を用いた変換テストが `scripts/verify_real_data.py` で即座に実行可能です。
- **ユニット/統合テスト**: 35件以上の網羅的なテストが含まれており、最新の Docling バージョン（v2.80.0等）にも完全対応しています。

## 5. 運用の容易さ

- **統一されたインターフェース**: PDFだけでなく、Officeドキュメント（DOCX, PPTX, XLSX）をすべて一つの関数 `process_pdf` で透過的に扱えます。
- **柔軟な設定**: 実行時にテーブル形式、数式抽出のオンオフ、OCRの挙動などを動的に変更可能です。
