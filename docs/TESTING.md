# テストの実施方法 (Testing Guide)

本プロジェクトでは、モジュールごとの詳細な挙動を確認する軽量なユニットテストと、実際のドキュメントレイアウトを検証する重いE2E (End-to-End) テストを分離して運用しています。

## 1. ユニットテストの実行 (高速)

環境構築時にインストールされる標準的な動作確認を行います。APIサーバーの振る舞い、Doclingの基本設定などをチェックします。
普段の開発サイクル（TDDなど）에서는こちらを利用します。

```bash
# すべての軽量テストを実行
uv run pytest
```

また、カバレッジを取得してどの程度テスト網羅されているかを確認する場合は以下のコマンドを使用します：

```bash
uv run pytest --cov=src
```

## 2. E2E (実データ) テストの実行 (重い処理)

Doclingの機械学習（VLMなど）パイプラインを使った本番同様のフル変換処理を行います。
実データの変換処理には時間がかかるため、**デフォルトの `pytest` コマンドからは除外**されています。リリース前の最終確認などに利用します。

### 2.1. テストデータの準備

まず、様々なファイルフォーマット (XLSX, PPTX, PDF, DOCX) の実データをダウンロードするスクリプトを実行します。

```bash
uv run python tests/download_samples.py
```
実行すると `tests/data/real_world` ディレクトリに各種サンプルファイルがダウンロードされます。

### 2.2. E2Eテストファイルの実行

重いテスト専用に分離した `e2e_real_world.py` を直接指定して `pytest` を実行します。

```bash
# 実データを用いた重いテストを実行
uv run pytest tests/e2e_real_world.py -v
```

GPUサポート (CUDA等) が利用可能な環境であれば自動的に高速化されますが、CPUのみの環境では非常に時間がかかる場合がありますのでご注意ください。

### 2.3. (代替) 検証スクリプトによる個別出力確認

よりシンプルにCLIとして出力結果（画像ファイルやMarkdownの出力状態）を確認したい場合は、専用の検証スクリプトを利用することも可能です。

```bash
uv run python scripts/verify_real_data.py
```
