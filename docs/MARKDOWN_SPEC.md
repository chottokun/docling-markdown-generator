# Markdown出力仕様 (Markdown Specification)

本ライブラリは、Doclingを用いて高度に構造化されたMarkdownを生成します。特に図（画像）の埋め込み、キャプションの保持、および複雑な表の変換において、LLMが理解しやすい形式を維持します。

## 0. メタデータ (YAML Frontmatter)

生成されるMarkdownの先頭には、ドキュメントのメタデータがYAML形式で付与されます。
```yaml
---
title: ドキュメントのタイトル
---
```
これにより、LLMは文書全体の文脈（どの文書を読んでいるか）を即座に把握できます。

## 1. 図（画像）の表現

### 埋め込み形式
図はMarkdown内で以下の形式で参照されます。
```markdown
![図のキャプション](images/image_hash.png)
```
- **保存場所**: 各変換リクエストごとに生成される一意のディレクトリ配下の `images/` フォルダに保存されます。
- **リンク**: 相対パスで記述されるため、ディレクトリごと移動しても整合性が保たれます。

### キャプションの抽出
Doclingの解析により、図に関連付けられたテキスト（図表番号や説明文）を可能な限りキャプションとして抽出します。

## 2. 表（テーブル）の表現

複雑なドキュメント（セルの結合があるPDFやExcelなど）に対応するため、表は **HTML (`<table>`) 形式** で Markdown 内に埋め込まれます。
- **高精度な構造保持**: `rowspan` や `colspan` を用いて、セルの結合を正確に維持します。
- **LLMフレンドリー**: 最近のLLMは Markdown 形式よりも HTML 形式のテーブルの方が、複雑なデータ構造を正確に理解できる特性があります。
- **アクセシビリティ**: 構造が維持されるため、LLMによるデータ抽出や分析（RAG等）において、列と行の関係を間違えることなく処理できます。

## 3. 文書構造

- **階層構造**: 見出し（# ## ###）が文書の論理構造に基づいて生成されます。
- **数式 (LaTeX)**: 科学技術文書に含まれる数式は、最新のVLMモデル（CodeFormulaV2等）を用いて高精度な **LaTeX 形式** で抽出・変換されます。これにより、LLMは数式の論理的意味を正確に解釈可能です。
- **リスト**: 箇条書きや番号付きリストが正確に再現されます。

## 4. サポートフォーマット
以下のファイルから同様の品質のMarkdownを生成可能です。
- **PDF**: 高精度なレイアウト解析。
- **Office (DOCX, PPTX)**: スタイルに基づいた構造化。
- **Excel (XLSX)**: シートごとのテーブル変換。
